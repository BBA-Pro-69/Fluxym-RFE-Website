🔵 Flux commerciaux — F2

Le Flux 2 est la branche commerciale du schéma en Y : la facture complète transite directement de la PA d'émission vers la PA de réception, sans passer par le PPF. C'est le flux visible par les entreprises — celui qui porte le document.

📨
Le Flux 2 transporte la facture dans son format natif (UBL, CII ou Factur-X) + la version convertie si le destinataire utilise un format différent.

🟠 Flux fiscaux — F1, F3, F12

📤

F1 — Données réglementaires

La PA-E extrait les données obligatoires de la facture et les transmet au PPF. C'est la branche fiscale du Y.

📥

F3 — Données complémentaires

Le PPF transmet à la PA-R les données dont elle a besoin pour informer le destinataire (si routage indirect).

🏛️

F12 — Vers la DGFIP

Le PPF agrège toutes les données collectées et les transmet à l'administration fiscale pour le pré-remplissage de la CA3.

🟣 Cycle de vie — F6

Le Flux 6 transporte les messages CDAR (statuts) dans les deux sens. Chaque changement de statut est notifié via F6 :

Déposée200PA-E → PPF✅ Oui
Reçue220PA-R → PA-ENon
Approuvée230PA-R → PA-ENon
Refusée210PA-R → PPF + PA-E✅ Oui
Encaissée212PA-R → PPF + PA-E✅ Oui
Rejetée213PA-R → PPF + PA-E✅ Oui

🟢 Annuaire — F7, F8, F9, F13, F14

5 flux dédiés à la gestion de l'annuaire central :

🟡 E-reporting & retours — F10, F11

F10 — E-reporting

  • F10.1 : Factures émises vers l'international
  • F10.2 : Factures reçues de l'international
  • F10.3 : Transactions B2C
  • F10.4 : Données de paiement (TVA encaissement)

F11 — Retours PPF

  • Acquittements de réception
  • Notifications de rejet (IRR, REJ)
  • Confirmations de traitement
  • Alertes et anomalies
